## 发现(Findings)

- **[设计注意 · HITL 与人 stage 的边界]** `stage_library.py` 中存在 `human_review` 这个 `StageCategory.HUMAN` 阶段。它代表"暂停等人"的 stage,与 invariant #6 不冲突——invariant #6 禁止的是把 `clarify`/`approval`/`supervisor` 这类**横切阻塞机制**误建模为 stage。`human_review` 是一个明确的、可编排的 stage,不是横切 concern。

- **[infra 范围]** invariant #5 当前仅锁定 `config.py` / `json_helpers.py` 两张**infra 叶子文件**。`infra/` 下其他模块(如 `db/models.py`、`terminal/ttyd.py`、`benchmarks/`)允许内部 import,这是现有架构设计,不是违规。

- **[re-export 的 fixture 注入]** 直接把 `test_gate_hard_fail` 的测试函数 import 到 invariants 后,pytest 在新目录下不会自动发现原文件里的 `quality_cfg` fixture。解决方式是在 `tests/invariants/conftest.py` 中显式定义同名 fixture,保持原文件不变即可兼容。
