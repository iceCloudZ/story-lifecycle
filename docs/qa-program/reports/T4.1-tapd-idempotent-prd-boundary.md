## 做了什么(What)

为 TAPD 接入层补了幂等性测试,为 PRD generator 补了空标题/超长标题边界测试:
- 在 `test_sync_by_id.py` 新增 `test_sync_by_id_is_idempotent`:同一 TAPD id 连续 sync 两次,
  第二次应为更新(`更新 1`)而非新建(`新建 0`),且 DB 中只保留一个 story。
- 在 `test_prd_generator.py` 新增:
  - `test_prd_generator_handles_empty_title`:空标题不崩,prompt 仍生成,LLM 结果正常返回。
  - `test_prd_generator_handles_very_long_title`:10000 字符标题直接进入 prompt,不崩、不丢内容。

未修改生产代码;现有 `sync_service.sync_tapd` 已用 `db.find_by_source_id` 做幂等检查,
PRD generator 本身对空/长标题也足够健壮。

## 发现(可选,Findings)

- TAPD sync 幂等性由 `sync_service.sync_tapd` 中的 `db.find_by_source_id` 保证,实现正确,只是缺测试。

- 测试用 CliRunner 走完整 CLI 路径,能覆盖命令行参数解析和输出文案,比直接调 service 函数更贴近真实使用。
