## 做了什么(What)

新建 `packages/story-lifecycle/tests/test_plan_confirm_pause.py`,验证 planner 的 plan/continue 暂停边界:

1. **暂停**:调用 `run_orchestrator_agent(...)` 完成规划后,断言 story context 中 `_plan_confirmed=False`,
   且 `subprocess.Popen` 未被调用(actions 未执行)。
2. **继续**:模拟用户 confirm(把 `_plan_confirmed` 置 `True`),调用 `continue_orchestrator_agent(...)`;
   通过预设 done file 让轮询立即完成,断言 `subprocess.Popen` 被调用(actions 执行),
   且 `_plan_confirmed=True` 仍保持在 context 中。

测试使用 `headless-smoke` profile 让 continue 走 headless 路径,并用 mock 替代真实 CLI 进程
及 supervise/kill 辅助函数,不启动真实 CLI。
