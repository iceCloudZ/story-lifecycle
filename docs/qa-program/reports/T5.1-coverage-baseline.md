## 发现(Findings)

- **[既有测试失败 / 与本卡无关]** `test_smoke.py::test_packaged_and_root_profiles_consistent` 失败。根目录 `packages/story-lifecycle/profiles/` 为空,而包内 `src/story_lifecycle/entry/profiles/` 有 6 个 yaml。原因待查(可能是 profiles 目录被误删或迁移)。
  - 影响:story-lifecycle 全量测试目前无法"全绿",但不影响 coverage 度量本身。
  - 建议后续:在后续某张卡(如 T4.x 或 T5.x 回归)修复该一致性,或在 CI 中同步检查。

- **[覆盖率盲区最大模块]** `sourcing` 模块仅 29.1%,其中 `sourcing/planner/` 和 `sourcing/workspace_diff.py` 几乎完全未覆盖;是后续补测试收益最大的区域。

- **[entry/cli 大面积未覆盖]** `entry/cli` 子模块 40.9%,大量交互式命令路径只在人工使用时触发,后续可通过 mock CLI 输入/输出补回归测试。

- **[infra/llm_client_kimi_cli.py]** 覆盖率 0%,属于特定 provider 集成代码,若长期无测试计划建议标注 `pragma: no cover` 或移至可选插件。
