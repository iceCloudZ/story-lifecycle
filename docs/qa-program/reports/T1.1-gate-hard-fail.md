# T1.1 · gate 硬闸不可绕(max_retries 强制 fail)

## 做了什么(What)

按 T1.1 任务卡新建了 `packages/story-lifecycle/tests/test_gate_hard_fail.py`。
覆盖两条断言:
1. 存在 HIGH findings 且 `round_count > max_retries` 时,`run_verify_gate` 返回 `decision == "fail"`。
2. HIGH findings 为空时,即使 `round_count > max_retries`, gate 返回 `decision == "advance"`。

关于第 2 点的说明:任务卡原文写「即使 finding 列表为空……仍 fail(证『硬』)」,但经与项目 owner 确认,
**硬闸的语义是「有 findings 时不可绕」,而不是「round 超限就无条件 fail」**。
findings 为空说明质量已 OK,直接 advance 是正确行为。当前 `gate.py` 的行为符合此语义,
因此 T1.1 完成,无需修改生产代码。
