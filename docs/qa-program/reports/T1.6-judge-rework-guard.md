## T1.6 修复细节

**原 bug**(`gate.py:229-238`):judge 判 rework 时 increment 后直接 return retry,无 max_retries 检查:
```python
if not verdict["pass"]:
    round_count = increment_review_round_count(context, stage)
    return {"decision": "retry", ...}   # ← 无超限检查,理论上能无限 retry
```

**修复**:increment 后先检查,与 HIGH-findings 路径(gate.py:247)同语义:
```python
if not verdict["pass"]:
    round_count = increment_review_round_count(context, stage)
    if round_count > max_retries:
        return {"decision": "fail", "reason": f"judge rework persists after {max_retries} repair rounds: ..."}
    return {"decision": "retry", ...}
```

**为什么这是对的**:gate 是硬闸(架构不变量 #2),两条 retry 路径(HIGH-findings + judge rework)必须**同等**受 max_retries 约束,否则 judge 路径绕过预算上限。T1.1 只覆盖了 HIGH-findings 路径,T1.6 补齐 judge 路径。
