## 做了什么(What)

新建 `packages/story-lifecycle/tests/test_launch_modes.py`,对同一 story/prompt 验证三种 claude
启动模式的产出意图:

1. **headless**:调用 `ClaudeAdapter.headless_launch_cmd`,断言返回 argv 使用 `claude -p`,
   并带 `--allowedTools` / `--permission-mode` 标志。
2. **interactive**:调用 `ClaudeAdapter.interactive_launch_cmd`,断言返回 argv 是 `claude "<prompt>"`
   形式且包含 prompt 文本。
3. **release**:调用 `generate_release_prompt`,断言返回 dict 的 `content` 是纯文本上线前检查清单,
   且包含 story 标题和 key。

三者都基于同一 `ClaudeAdapter`,确保后续改启动方式时能通过此测试发现行为漂移。

## 发现(Findings)

- **[设计说明]** headless 模式(`-p`)的 prompt 不放在 argv 中,而是由调用方通过 stdin 写入
  (`continue_orchestrator_agent` 中 `headless_proc.stdin.write(cli_prompt.encode(...))`)。
  因此"三者都含 prompt"的断言在 headless 上体现为 argv 含 `-p` 标志(表明 prompt 经 stdin 注入),
  而不是 prompt 字符串本身。这是当前设计的正常行为,不是 bug。
