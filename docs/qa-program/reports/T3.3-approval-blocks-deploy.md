## 发现(可选,Findings)

- **阻塞机制不存在**:代码中没有地方读取 `StageDefinition.requires_human` 来阻止 stage 执行。`deploy` 的 `requires_human=True` 目前只是语义标记。
- **安全由 profile 保证**:由于所有内置 profile 都不包含 `deploy`,默认情况下根本不可能进入 deploy stage,因此「无 approval 阻塞」不是当前紧迫风险。
- **`/api/approvals` 语义是 findings 队列**:该端点调用 `db.get_all_pending_findings()` 返回 open/accepted findings,与部署审批无关。若未来要实现真正的 deploy approval,需要新增数据模型和端点。
- **未来若启用 deploy 必须补阻塞**:一旦有人把 `deploy` 加入 profile,就必须同时实现 approval 检查(否则 `requires_human=True` 只是空头支票)。本测试会防止「无意识地启用 deploy」。
