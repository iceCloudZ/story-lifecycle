## 做了什么(What)

为 supervisor 的 HITL 决策补了边界测试,并修复了 `handle_pty_output` 在 LLM 返回非法格式时会抛异常崩溃的缺口:
- 新增 `test_supervisor_boundary.py` 覆盖:
  - mock LLM 返回标准 `{choice, reason}` → `decide_response` 正确解析。
  - PTY buffer 命中提问 marker → `handle_pty_output` 触发决策、写 PTY、落 log。
  - LLM 返回非法 JSON / choice 不在 options / 缺字段 → `handle_pty_output` 降级返回 `False`,无副作用。
- 修改 `supervisor.py` 的 `handle_pty_output`:在调用 `decide_response` 外加 `try/except`,捕获任何决策异常后记录 warning 并返回 `False`,避免单个 LLM 失常回答炸掉整个 PTY session。

## 发现(可选,Findings)

- `supervisor.py` 原 `handle_pty_output` 未处理 `decide_response` 抛出的 `JSONDecodeError`/`KeyError`/`ValueError`,LLM 一旦返回非法格式就会导致整个 PTY 监督循环崩溃。这是真实风险,已修复。
- 修复位置严格限定在 Handler(`handle_pty_output`),Pure Decider(`decide_response`) 仍保持原语义(非法输入继续抛异常,由 Handler 捕获),符合架构不变量「Decider 纯、Handler 管副作用/降级」。
- `log.warning` 使用 `__name__` logger,测试时不会输出到 stdout,不影响 pytest 输出可读性。
